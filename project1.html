Description

For this project,I was in team with Jérôme Sartiaux, Loïc Tuplink, Antoine Sprumont, Bastien Igot, Léandre Karasavas. We had to make a game with two dice. The principle of the game was simple, we started with some points for everyone. Then the player lunch "two dice", one player by one. The result of the two dice was sum. After it subtracts the point by the result down to 0, the first player at 0 wins. But we put one obstacle if we go down 0, it didn't subtract and stay like before the player lunched.

The project was separate in two parts : an electronique part and an It part. This two part were connected by a communication card.

The electronique part had 14 leds, numbered from 1 to 7 by each part. Each one represent the point of the dice like you see in the scematics. Here is the logical for each led on :

  • 1 : D7
  • 2 : D5 D6
  • 3 : D7 D5 D6
  • 4 : D1 D2 D3 D4
  • 5 : D7 D1 D2 D3 D4
  • 6 : D5 D6 D3 D4 D1 D2

The electronique part had 4 different integrated circuit:

CD4093

The CD4093 are composed by 4 logic gate NAND, each gates switched at different points for positive and negative signals, the signal out was reversed (High or Low)

CD4069

The CD4069 his jobs was to reversed the polarisation of the signal
(High => Low or Low => High)

CD4029

The CD4029 is an up/down counter. it can count in binary or decimal, and relunch the count when it can't go higher.

CD4053

The CD4053 is an analogue multiplexer/demultiplexer triple. Each part had 2 canals and 3 entry command


Here was the logic of the programs :

When it was lunch, a loop makes a check to see if the points different to 0 if not. It listened to the communication card and took the two dice and did the sum after substract the points by the sum. It did an other verification, if the sum was bigger or equals to 0 , the points are authorized. If not the point stayed the same.
